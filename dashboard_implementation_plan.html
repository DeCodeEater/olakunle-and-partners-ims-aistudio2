Phase 4 — Design system & edge cases
Establish foundations that prevent future inconsistency · Estimated: 1 week
Prevents regression
Define and document a color token system
Create a named color system with semantic tokens — not raw hex values used ad hoc. This prevents the rainbow-color problem from recurring.
Design ~3 hrs
Minimum token set to define:
· --color-primary: your navy (used for: bars, active nav, primary buttons, chart series 1)
· --color-secondary: your green (used for: positive trends, success states, chart series 2)
· --color-danger: red (used for: over-budget, urgent lease, negative trend, error states)
· --color-warning: amber (used for: below-target, approaching threshold, pending states)
· --color-neutral: grey (used for: inactive bars, secondary chart series, disabled states)
· --color-surface: card background · --color-border: card border

Document: "A chart bar is always --color-primary unless a semantic condition overrides it." Write this rule down and share it with anyone who builds UI.
Define a typography scale with 4 levels
Currently text sizes are inconsistent between sections. Establish a fixed scale and apply it throughout.
Design ~2 hrs
4-level type scale:
· Hero: 32px, weight 500 — KPI headline numbers (₦342.1M, 8.4%)
· Section title: 16px, weight 500 — "Financial Performance", "Recent Expenses"
· Body / label: 13px, weight 400 — descriptions, chart labels, nav items
· Metadata: 11px, weight 400, muted color — timestamps, subtitles, footnotes

Rule: Never use all-caps below 12px. Never use 6 different sizes in one card. Apply scale as CSS variables or Figma text styles — whatever your tooling supports.
Design loading skeleton states for all cards
Every KPI card, chart, and the Transactions Hub needs a skeleton loader for when data is fetching. Without this, production looks broken on slow connections.
Design Code ~3 hrs
1. Create grey shimmer blocks that mirror the shape of each card's content.
2. KPI card skeleton: one large block (number area) + two small blocks (label + trend).
3. Chart skeleton: a series of grey bars or a wavy grey line in the chart area + axis placeholder lines.
4. Transactions Hub skeleton: 3 rows of [dot] [two lines of text] [tag].
5. Animate with a shimmer sweep (CSS animation moving a gradient left-to-right).
6. All skeletons should match the exact dimensions of the loaded card to prevent layout shift.
Design empty states for key sections
What does the dashboard look like for a new account with no data? Or when the Lease Timeline has no upcoming expirations? These states need designs.
Design ~2 hrs
Priority empty states to design:
· Lease Timeline: "No leases expiring in the next 7 months — add a property to get started" + CTA button.
· Transactions Hub: "No recent transactions — your activity will appear here."
· Live Actions: "All caught up — no new activity."
· Facilitator Performance: "No facilitators assigned — invite your team."

Each empty state: small illustration/icon + one-line message + optional CTA. Keep tone informative, not apologetic.
Design error states and retry patterns
When a data API fails, a card should show an error message with a retry button — not a blank card or a frozen loader.
Design Code ~1.5 hrs
1. Error card: red border-left accent, warning icon, "Could not load [section name]" message, "Retry" button.
2. Global error banner: if multiple sections fail simultaneously, show one top-of-page banner: "Some data could not be loaded. [Retry all]"
3. Partial error: if only one chart fails, show the error inline without disrupting surrounding cards.
4. Include last-known-good data with a warning: "Showing data from 3h ago — live data unavailable."

Phase 5 — Polish & accessibility
Final quality pass before production · Estimated: 3–5 days
Before launch
Run WCAG AA contrast audit on all text
Check every text element against its background. Small labels and muted metadata text are likely failing the 4.5:1 minimum ratio.
Design ~2 hrs
Tools: Figma Contrast plugin, or Chrome DevTools Accessibility panel, or https://webaim.org/resources/contrastchecker/

Common failures to check:
· "YTD Recognized", "Lease Efficiency", "Active Contracts" labels — likely too light
· Chart axis labels (year labels 2020–2024) — likely failing at that size
· "VOL: ₦3,428 AVG" — small text that may fail
· Muted delta values (↑2.1%, ↑12.4%) — check the green against white background

Fix by darkening text color, not by making backgrounds lighter.
Upgrade sidebar active and hover states
Active item needs a left-border accent + pill background. Sub-items need visual indent. Collapsed mode should be possible for power users wanting more content space.
Design UX ~2 hrs
Active state: 2px left-border in brand navy + very light navy background pill (#E6F1FB equivalent).
Hover state: light grey background with smooth 100ms transition.
Sub-items (Landlords, Tenants, Buyers, Facilitators; Commissions, Taxes, Financial Reports): 12px left indent, 12px font size instead of 13px.
Collapsed mode: clicking the hamburger (≡) collapses sidebar to icon-only (48px wide). Icons remain, labels hide. Tooltip on hover shows label.
Add a subtle divider line between MAIN / PEOPLE / FINANCE sections (currently uses text labels only).
